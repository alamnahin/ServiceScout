# ServiceScout

ServiceScout is a React + TypeScript marketplace for booking local services. It includes client, provider, and admin experiences, Supabase-backed data, secure booking flows, and PayWay payment support.

## Overview

The app is built for service discovery and booking with separate flows for customers, providers, and administrators. Customers can browse services, read service detail pages, book appointments, and pay securely. Providers can manage bookings, services, earnings, and schedules. Administrators can manage categories, services, providers, disputes, withdrawals, and platform settings.

## Tech Stack

- React 18
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui and Radix UI
- Supabase
- React Router
- Framer Motion
- Recharts
- Sonner for toasts

## Key Features

- Public landing pages, About page, and Terms page
- Service browsing with search and category filters
- Rich service detail pages with markdown rendering
- Booking flow with dynamic duration pricing
- Secure payment integration with PayWay
- Customer dashboard, provider dashboard, and admin dashboard
- Analytics charts for bookings, earnings, providers, and platform activity
- Withdrawals, disputes, reviews, and service management

## Getting Started

### Prerequisites

- Node.js 18 or newer
- pnpm
- A Supabase project
- PayWay credentials if you want to use real payments

### Install

```bash
pnpm install
```

### Environment Variables

Create a `.env` file in the project root with the following values:

```env
VITE_SUPABASE_URL=
VITE_SUPABASE_PUBLISHABLE_KEY=
VITE_PAYWAY_PUBLISHABLE_KEY=
VITE_USE_MOCK_PAYWAY=true

# Used by the server route under api/payway/[action].ts
PAYWAY_MERCHANT_ID=
PAYWAY_API_URL=https://https://api.payway.com.au/rest/v1/transactions
PAYWAY_PUBLIC_KEY=
PAYWAY_RSA_PUBLIC_KEY=
PAYWAY_RETURN_URL=
PAYWAY_CANCEL_URL=
```

Notes:

- `VITE_USE_MOCK_PAYWAY=true` keeps payment flows in mock mode for development.
- Set `VITE_USE_MOCK_PAYWAY=false` when you want the frontend to call the server PayWay route.
- The Supabase client will fail fast if `VITE_SUPABASE_URL` or `VITE_SUPABASE_PUBLISHABLE_KEY` is missing.
- The PayWay server route requires `PAYWAY_MERCHANT_ID`, `PAYWAY_API_URL`, and `PAYWAY_PUBLIC_KEY`.

### Run Locally

```bash
pnpm dev
```

## Payments

The payment flow consists of three main parts:

1. **PayWay Card Widget** (client-side): Securely tokenizes the credit card
2. **Supabase Edge Function** (`process-payment`): Processes the token and calls PayWay API
3. **Booking Creation** (client-side): Creates the booking record after successful payment

### Payment Modes

- `VITE_USE_MOCK_PAYWAY=true` (default): Uses mock PayWay responses for development
- `VITE_USE_MOCK_PAYWAY=false`: Calls real PayWay API (requires credentials)

### Deploying the Edge Function

The Supabase Edge Function `supabase/functions/process-payment/index.ts` must be deployed to your Supabase project:

```bash
# Install Supabase CLI
npm install -g supabase

# Login to Supabase
supabase login

# Deploy the edge function
supabase functions deploy process-payment --project-id YOUR_PROJECT_ID
```

### Development

For local development with mock payments:
- No additional setup needed; payments will use mock responses
- The edge function will respond immediately with mock transaction data

### Production

For real payments:
1. Set `VITE_USE_MOCK_PAYWAY=false` in your environment variables
2. Configure PayWay credentials in Supabase project settings
3. Deploy the edge function to your production Supabase project
4. Set `PAYWAY_MERCHANT_ID` and other credentials in the Supabase Edge Function environment

## Supabase

The app expects a configured Supabase backend with the project schema and migrations in place. Apply the SQL migrations in `supabase/migrations` before testing production flows.

## Deployment Notes

- Build output is generated by Vite in `dist/`
- The app uses a server route under `api/payway/[action].ts`, so deploy it on a platform that supports API/serverless routes
- Set production environment variables for Supabase and PayWay before enabling real payments
